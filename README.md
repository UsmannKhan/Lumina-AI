# Lumina

> Turn anything you see into something you remember.

Lumina is an AI-powered study companion. Drop in a **YouTube video**, **PDF**, **audio recording**, **Word doc**, **PowerPoint**, or **any web article**, and it turns the source into a complete study set: structured notes, flashcards, quizzes, and (for programming content) practice problems. Organize related sources into **Spaces** — like a course or a research project — and study everything together.

---

## Features

### Source types
| Source | What Lumina does |
|---|---|
| **YouTube** | Pulls the transcript with timestamps, generates timestamped notes and a "Key Concepts" timeline |
| **PDF** | Uploads to storage, runs Gemini vision over the full PDF so figures, tables, and equations all inform the notes |
| **Audio** (mp3, wav, m4a, ogg, webm, flac, up to 25 MB) | Transcribes the audio and produces notes in one pass; player with waveform visualizer included |
| **Webpages** | Scrapes the article content; recognizes arXiv URLs and auto-routes to the PDF for figure-aware notes |
| **DOCX / PPTX** | Converts to PDF via Gotenberg, then treats it like a PDF source |
| **TXT** | Direct text ingestion |

### Study modes (per source)
- **Notes** — Markdown with sections, tables, Mermaid diagrams, KaTeX math; editable "My Notes" tab for manual additions
- **Ask** — chat with the source. Optional web search augmentation
- **Flashcards** — generated with topic/difficulty/count controls; multiple named sets per source; spaced manual review with hint/explanation/timestamp links
- **Quiz** — MCQ, true/false, and short-answer mix; AI-graded short answers; per-quiz score history
- **Practice** — for CS sources, autogenerated coding problems with Monaco editor, AI feedback, and submission tracking

### Spaces (collections)
- Group related sources for a course, project, or topic
- Action cards above sources: **Review flashcards** and **Take a quiz** aggregate every flashcard set and quiz across all sources in the space
- Click any set or quiz row → opens the source chat directly in that tab, with that set/quiz already running

### Other
- Email/username/password auth with JWT
- Source-content-aware chat memory per chat
- Sidebar with recent chats, pinned spaces, search across sources
- Export notes to PDF
- Auto-cleanup of Supabase Storage when sources or accounts are deleted
- Rate limiting on AI endpoints

## Tech stack

| Layer | Stack |
|---|---|
| **Frontend** | Next.js 15, React 19, TypeScript, Tailwind CSS 4 |
| **UI** | Tiptap (rich text), Monaco (code editor), React-PDF, Mermaid, KaTeX, Lucide icons |
| **Backend** | FastAPI, SQLAlchemy, slowapi (rate limiting), python-jose (JWT), bcrypt |
| **Database** | PostgreSQL via Supabase |
| **Storage** | Supabase Storage (separate `pdfs` and `audio` buckets) |
| **AI** | Google Gemini (`gemini-3.1-pro-preview` for notes; `gemini-3-flash-preview` for chat, flashcards, quiz, grading) |
| **External services** | YouTube transcript via RapidAPI; DOCX/PPTX conversion via Gotenberg |

## Setup

### Prerequisites

- **Python 3.11+** with `pip` and `venv`
- **Node.js 20+** with `npm`
- A **Supabase** project (free tier is fine)
- A **Google AI Studio** API key (Gemini access)
- A **RapidAPI** account with subscription to a YouTube transcript API
- *(Optional)* A **Gotenberg** instance for DOCX/PPTX conversion (Docker run is simplest)

### 1. Clone and set up Supabase

Create a Supabase project, then:

**Database tables** — let SQLAlchemy create them automatically on first backend start, or run migrations manually. See `app_backend/api/models.py` for the schema.

**Storage buckets** — create two buckets in the Supabase dashboard:

| Bucket name | Allowed MIME types | File size limit |
|---|---|---|
| `pdfs` | `application/pdf` | 10 MB |
| `audio` | `audio/*` (or explicit list: `audio/mpeg, audio/wav, audio/mp4, audio/ogg, audio/webm, audio/flac`) | 25 MB |

Both buckets should be private (the backend issues signed URLs for access).

### 2. Backend

```bash
cd app_backend
python -m venv venv
# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

pip install -r requirements.txt
```

Create `app_backend/.env`:

```env
# PostgreSQL — Supabase connection string (Project Settings → Database → Connection string)
DATABASE_URL=postgresql://postgres:<password>@<project>.supabase.co:5432/postgres

# Supabase Storage — service key from Project Settings → API
SUPABASE_URL=https://<project>.supabase.co
SUPABASE_SERVICE_KEY=<service_role_key>

# Auth — generate any long random string for JWT signing
AUTH_SECRET_KEY=<random_64_char_string>
AUTH_ALGORITHM=HS256

# Google Gemini
GOOGLE_KEY=<google_ai_studio_api_key>

# YouTube transcript API (subscribe to one on RapidAPI, e.g. "YouTube Transcripts")
RAPIDAPI_KEY=<rapidapi_key>
RAPIDAPI_HOST=<host_from_rapidapi_dashboard>

# Optional — DOCX/PPTX → PDF conversion via Gotenberg
GOTENBERG_URL=http://localhost:3000
```

Run the backend:

```bash
uvicorn api.main:app --reload --port 8000
```

API docs auto-served at `http://localhost:8000/docs`.

### 3. Frontend

```bash
cd app_frontend
npm install
```

Create `app_frontend/.env.local`:

```env
NEXT_PUBLIC_API_URL=http://localhost:8000
```

Run the frontend:

```bash
npm run dev
```

Open `http://localhost:3000`.

### 4. (Optional) Gotenberg for DOCX/PPTX

Easiest is Docker:

```bash
docker run --rm -p 3000:3000 gotenberg/gotenberg:8
```

Point `GOTENBERG_URL` in the backend `.env` at it. If you don't run Gotenberg, DOCX/PPTX uploads will fail with a clear error — PDF / TXT / audio / YouTube / website work without it.

## Architecture notes

### Per-source AI flow
Source ingestion is single-pass: the source content (transcript, PDF bytes, audio bytes, scraped HTML) is sent to Gemini with a depth-controlled prompt that produces both the notes and a delimited "Key Concepts" section. A post-processor parses out the key concepts into structured rows and stores them. Flashcards, quizzes, and code problems are generated on demand by separate prompts using the same source content + selected key concepts as context.

### Storage
PDFs and audio files live in Supabase Storage under `{user_id}/{uuid}.{ext}`. Signed URLs are issued on demand for the viewer/player. Deletion cascades — removing a chat or a user removes their Storage objects.

### Auth
JWT bearer tokens from a standard OAuth2 password grant. Tokens stored in `localStorage`. Frontend dispatches a custom event on 401 so the auth context can drop users back to login.

### Markdown rendering
Notes and AI chat replies are rendered through a single `MarkdownRenderer` component with `remark-gfm`, Mermaid embedding, KaTeX support, and a defensive post-pass that repairs collapsed-single-line tables (an occasional LLM output quirk).
